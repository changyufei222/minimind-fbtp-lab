# Contributing

Thank you for helping improve this repository.

## Good contributions

- Reproducible bug reports with the exact command, environment, and smallest relevant input
- Documentation fixes and clearer setup instructions
- Tests for confirmed behavior
- Data corrections that cite the affected record, field, and supporting source
- Small, focused pull requests that preserve the repository's stated scope

## Before opening a pull request

1. Search existing issues and pull requests.
2. Do not commit API keys, credentials, private datasets, patient information, or local absolute paths.
3. Keep generated outputs separate from source code and state how they were produced.
4. Run the relevant local tests or validation scripts.
5. Explain the scientific or engineering impact of the change.

## Scientific corrections

For a data or analysis correction, include:

- affected file and record identifier
- current value and proposed value
- evidence source
- whether downstream tables, figures, or reports are affected

## Scope

This is a research and portfolio repository. Large feature proposals should begin as an issue so that data provenance, reproducibility, and cross-repository effects can be reviewed first.
